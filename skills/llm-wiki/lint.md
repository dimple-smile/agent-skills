# llm-wiki:lint

健康检查 Wiki，检测矛盾、孤儿页面、缺失概念等问题，保持 Wiki 长期健康。

## When to Use

- 用户说"检查一下 wiki"、"帮我看看有没有问题"、"整理一下知识库"
- Wiki 积累到一定规模后（建议 20+ 页面时定期执行）
- 每次摄入一批重要资料后

## 检查项

### 1. 矛盾检测

扫描不同页面间的事实冲突：
- 同一实体在不同页面的描述不一致
- 同一概念在不同来源中的定义冲突
- 标注为矛盾但尚未解决的内容

检查方法：读取所有实体页和概念页，对比相同主题的描述。

### 2. 过时信息

识别被新来源取代的旧声明：
- 页面的 `updated` 日期远早于相关来源的日期
- 有新来源明确否定了旧结论

### 3. 孤儿页面

找出没有任何页面通过 `[[wikilink]]` 链接到的页面。

检查方法：扫描所有页面中的 `[[wikilink]]`，统计每个页面的入站链接数，入站为 0 的即为孤儿。

### 4. 缺失页面

被 `[[wikilink]]` 引用但尚未创建的页面。

检查方法：收集所有 `[[wikilink]]` 引用，对比实际存在的文件列表。

### 5. 缺失交叉引用

应该在概念上关联但未建立链接的页面。

检查方法：分析相同来源引用的页面，如果两个页面共享 2 个以上来源，可能应该互相链接。

### 6. 数据缺口

可以通过搜索补充的信息空白。

检查方法：识别概念页中列出的"开放问题"，以及概览中尚未深入的方向。

## 工作流

### Step 1: 读取全貌

```bash
# 获取所有 Wiki 页面列表
ls -R wiki/
# 读取目录
cat wiki/index.md
```

### Step 2: 逐项检查

按上述 6 个检查项依次扫描。对于大型 Wiki，可以分批检查：

```
正在检查 Wiki 健康...
  [1/6] 矛盾检测... ✅ 发现 2 个问题
  [2/6] 过时信息... ✅ 发现 1 个问题
  [3/6] 孤儿页面... ✅ 发现 3 个页面
  [4/6] 缺失页面... ✅ 发现 4 个未创建页面
  [5/6] 缺失交叉引用... ✅ 发现 5 组建议关联
  [6/6] 数据缺口... ✅ 发现 2 个信息空白
```

### Step 3: 生成报告

```markdown
## Wiki 健康报告 - YYYY-MM-DD

### 统计
- 总页面数：X（实体 X / 概念 X / 主题 X / 来源 X）
- 来源数：X
- 平均每页面交叉引用：X
- 最近更新：YYYY-MM-DD

### 发现的问题

#### ⚠️ 矛盾（2）
1. [[page-a]] 与 [[page-b]] 关于 X 的描述不一致
2. [[page-c]] 中的 Y 声明被 [[source-new]] 否定

#### 🕐 过时信息（1）
1. [[page-d]] 最后更新于 YYYY-MM-DD，相关来源已有更新

#### 🔗 孤儿页面（3）
1. [[page-e]] - 0 个入站链接
2. [[page-f]] - 0 个入站链接
3. [[page-g]] - 0 个入站链接

#### 📝 缺失页面（4）
1. [[concept-h]] - 被引用 3 次但未创建
2. [[concept-i]] - 被引用 2 次但未创建
3. [[entity-j]] - 被引用 2 次但未创建
4. [[concept-k]] - 被引用 1 次但未创建

#### ↔️ 缺失交叉引用（5）
1. [[page-a]] ↔ [[page-b]] - 共享 3 个来源但未互相链接
2. ...

#### 🔍 数据缺口（2）
1. 主题 X 缺少关于 Y 的资料
2. [[concept-h]] 的开放问题尚未解决

### 建议操作（按优先级）
1. [高] 解决 page-a 与 page-b 的矛盾
2. [高] 创建 [[concept-h]]（被引用 3 次）
3. [中] 为孤儿页面添加入站链接
4. [中] 更新 page-d 的过时信息
5. [低] 搜索补充关于 Y 的资料
```

### Step 4: 询问用户是否自动修复

```
发现 X 个问题。要我自动修复吗？

可以自动修复的：
  - 创建缺失页面（需要我根据上下文推断内容）
  - 添加缺失的交叉引用
  - 解决部分矛盾（需要你确认保留哪个版本）

需要你确认的：
  - 过时信息的更新方式
  - 矛盾的解决方案
```

### Step 5: 执行修复（如用户同意）

根据用户选择，执行修复操作。每项修复后更新 `wiki/log.md`。

```markdown
## [YYYY-MM-DD] lint | Wiki 健康检查

- **问题总数**：X
- **已修复**：X
- **待确认**：X
- **修复详情**：
  - 创建缺失页面：[[concept-h]], [[concept-i]]
  - 添加交叉引用：[[page-a]] ↔ [[page-b]]
  - 标记矛盾：[[page-a]] vs [[page-b]]
```
